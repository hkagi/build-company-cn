# Skill: agicompany

## 功能介绍
快速搭建智能体公司，完成6大核心工作：
1. **战略目标**——注灵魂（去哪）
2. **组织设计**——搭骨架（分工）
3. **岗位梳理**——填肌肉（发力）
4. **制度建设**——连神经（贯通）
5. **权责配置**——定筋骨（支撑）
6. **文化激励**——注血液（激活）

## 文件目录
1. 各部门的共用工作区在 openclaw 工作目录下的 workspace/agents 目录下的部门子目录中，称为部门目录。如总经办的部门目录为：~/.openclaw/workspace/agents/GM/
2. 各部门员工的个人工作区在部门子目录下的个人子目录中，称为个人目录。主总经办秘书 snow 的部门目录为：~/.openclaw/workspace/agents/GM/snow
3. 部门目录下有 AGENTS.md，IDENTITY.md，SOUL.md，RULES.md 文件，内容分别是工作指南、部门身份、部门灵魂、工作规范，所有部门员工每天工作前都要先阅读 AGENTS.md，并根据指引阅读其它部门文件。
4. 部门目录下还有各个岗位的职责、要求，以 JOB_ 开头，如 JOB_UI_DESIGNER.md 里写了 UI 设计师的职责和要求。
5. **部门目录下的 AGENTS.md，IDENTITY.md，SOUL.md，RULES.md 须参考本 skill 附带的同名文件**。
6. 个人目录下有 AGENTS.md，IDENTITY.md，SOUL.md，TOOLS.md，USER.md，HEARTBEAT.md 文件，遵照 openclaw 的通常定义和使用方法。
   - AGENTS.md 要有如下要求：工作前须阅读上一级目录（部门目录）中的 AGENTS.md 文件。
   - IDENTITY.md 要有如下要求：根据自己的岗位阅读部门目录中对应岗位的职责、要求。

## 使用方法
1. 问公司名，（提示：如香港通用智能有限公司），默认：我的公司
2. 问公司主营，默认：什么都想做
3. 问各个部门名，默认：技术部、设计部、销售部、运营部、总经办
4. 逐个部门询问技术岗位（每个部门给出3到5个默认岗位）
5. 办理入职，每个岗位询问是否招人，招人的话，输入中文名、英文名，确认后仍问这个岗位是否招人，选否进入下一个岗位。
6. 创建完成后仍可以再增加部门、岗位，办理新入职。

## 程序实现
1. 先不做实际动作，只是记录用户的输入。
2. 实际操作前向用户展示拟订的组织架构图及各部门员工。
3. 操作时按上文的文件目录设计建议目录和文件，所有的 md 都要根据经验填上必要的信息（比 JOB_UI_DESIGNER.md 里由程序生成 UI 设计师的职责和要求，各部门的公用 md 和各人的各个默认 md 都要按 openclaw 默认结合上文要求填好）。
4. openclaw 里增加新人的流程不要漏掉，比如在 ~/.openclaw/openclaw.json 中增加新员工，配置正确的个人工作目录。
5. 部门目录下的 AGENTS.md，IDENTITY.md，SOUL.md，RULES.md 须分别参考本 skill 附带的同名文件。
